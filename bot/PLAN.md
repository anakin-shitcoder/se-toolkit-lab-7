# Telegram Bot Development Plan

## Task 1: Scaffold (Current Phase)

Создание базовой структуры бота с чётким разделением ответственности между компонентами. Архитектура основана на принципе разделения логики обработки команд и Telegram-специфичного кода. Обработчики команд (handlers) реализованы как чистые функции, которые принимают строку команды и возвращают текстовый ответ. Это позволяет тестировать логику без запуска Telegram-бота и без зависимости от библиотеки aiogram в тестах. Поддержка флага `--test` в точке входа позволяет запускать отдельные команды и проверять их работу через stdout.

## Task 2: Backend Integration

Интеграция с LMS (Learning Management System) backend для получения данных о студентах, лабораторных работах и оценках. Будет создан `LMSClient` в модуле `services/`, который использует `httpx` для асинхронных HTTP-запросов к API. Клиент будет обрабатывать аутентификацию через API-ключи, retry-логику для устойчивости к временным ошибкам сети, и кэширование ответов для снижения нагрузки на backend. Обработчики команд `labs` и `scores` будут использовать этот клиент для получения реальных данных вместо заглушек.

## Task 3: Intent Routing with LLM

Добавление поддержки естественного языка через интеграцию с LLM (Large Language Model). Модуль `llm_client.py` будет отправлять пользовательские сообщения в LLM API для определения интента (намерения). Например, запрос "как мои дела?" будет распознан как запрос оценок и перенаправлен в обработчик `scores`. Это позволит пользователям взаимодействовать с ботом в свободной форме, без необходимости запоминать конкретные команды. Будет реализована система fallback: если LLM недоступен, бот переключается на режим точных команд.

## Task 4: Deployment

Развёртывание бота в production-среде с использованием Docker. Будет создан `Dockerfile` для контейнеризации приложения и `docker-compose.yml` для оркестрации с зависимостями. Конфигурация через переменные окружения с поддержкой `.env.bot.secret` для чувствительных данных (токен бота, API-ключи). Предусмотрена поддержка health checks для мониторинга доступности бота, логирование через structured logs для отладки, и graceful shutdown для корректного завершения работы при обновлении. Будет настроен CI/CD пайплайн для автоматического деплоя при изменениях в репозитории.
